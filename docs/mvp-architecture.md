# Architecture fonctionnelle du MVP

## 1. Entrée publique
- Landing page
- Présentation courte de la promesse
- CTA principal : **Faire le diagnostic gratuit**
- CTA secondaire : **Explorer les ressources**

## 2. Diagnostic gratuit
- questionnaire multi-étapes très court (10 à 15 questions)
- progression visible
- sauvegarde simple de session
- consentement minimal / RGPD clair

## 3. Restitution personnalisée
- profil principal
- 3 fragilités prioritaires
- 3 recommandations concrètes
- 3 ressources associées
- CTA de capture : email pour recevoir le résultat complet / continuer

## 4. Bibliothèque de ressources
- fiches pratiques
- FAQ métier
- trames / modèles
- filtres par thème, niveau d'expérience, urgence, type de projet

## 5. Espace conversationnel spécialisé (MVP léger)
- interface simple type question/réponse
- prompt system spécialisé résidents locaux
- garde-fous : informations générales, non substitutives à un avis juridique
- suggestions de questions fréquentes

## 6. Segmentation CRM simple
- score de maturité
- niveau d'isolement
- besoin principal
- thème prioritaire
- niveau d'intention (gratuit / premium / humain)

## 7. Offres freemium / premium
### Gratuit
- diagnostic
- restitution synthétique
- quelques ressources gratuites
- inscription email

### Premium léger
- ressources premium
- parcours thématiques
- veille ciblée
- accès élargi au copilote conversationnel

### Premium accompagné
- visio de cadrage
- orientation experte
- accompagnement ponctuel

## 8. Back-office minimal
- gestion des contenus
- consultation des leads et segments
- ajout / édition des ressources
- export CSV simple
